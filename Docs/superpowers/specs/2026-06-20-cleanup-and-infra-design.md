# Cleanup & Infrastructure Design

**Date:** 2026-06-20
**Branch:** `looped`
**Status:** Approved (design phase)

## Purpose

Pico-DSP-Garden has accumulated organizational debt: a broken `.gitignore`,
committed build artifacts and IDE state, a split-but-uncommitted `AGENTS/`
directory, an AGENTS claim ("no host build, no test framework, no CI") that we
are about to make false, and three design docs that reference ~9 classes which
do not exist in the code. This spec covers two efforts:

1. **Cleanup** — repo hygiene, doc reconciliation, committed-and-correct
   agent guidance.
2. **Infrastructure** — host test harness (CMake + doctest), a build-all-examples
   command, and CI to run both.

Cleanup is a hard prerequisite for infrastructure: the test/build/CI work lands
on a clean, honest foundation, and no file is touched twice.

## Non-goals

- No changes to DSP algorithm behavior. `libraries/rpdsp` and
  `libraries/pico_audio_i2s` source is untouched.
- No new DSP or platform abstractions. The aspirational block framework
  (`DefaultAudioBlock`, `I2sSampleBridge`, etc.) is **not** being implemented;
  it moves to roadmap.
- No changes to example sketches' DSP logic.
- No on-device flashing in CI. Verification of audio behavior remains
  "flash and listen," done manually.

## Branching

All work proceeds on `looped` (current branch). The implementation plan will
define commit granularity; the expectation is one commit per logical step
(e.g. hygiene, each doc, the test harness, the build script, CI) rather than
one large commit.

### 1.6 Erase the false "no host build" claim

>   invent test/lint/typecheck commands; none exist. Verification = *compile the
>   sketch, flash, listen.*

**Replacement:**
> - **Host tests live in `tests/`** (CMake + doctest). Build and run with

>   **Firmware builds via `arduino-cli`** — `scripts/build_all_examples.sh`

>   Verification of audio behavior is still "flash and listen"; the host suite

### 2.1 Host test harness — CMake + doctest

**`tests/CMakeLists.txt`:**
- C++17, `cmake_minimum_required(VERSION 3.16)`.
- `target_include_directories(... libraries/rpdsp/src)` so
  `#include <rpdsp/oscillator.h>` resolves identically to how examples resolve it.
- Compiles `main.cpp` (doctest registration with `DOCTEST_CONFIG_IMPLEMENT`)
  and the test files into one test binary named `rpdsp_tests`.
- `enable_testing()` + `add_test(NAME rpdsp_tests COMMAND rpdsp_tests)`.
- Links against the existing host-dummy shim in `hardware_interpolator.h`
  (already present, guarded by `__has_include(<hardware/interp.h>)` — no new
  shim code needed).

**`tests/doctest.h`:** vendored single-header (MIT). Keeps the host build
hermetic — no vcpkg/conan/system dependency. Pinned to a tagged release; version
recorded in a comment at the top.

**`tests/main.cpp`:** minimal — `#define DOCTEST_CONFIG_IMPLEMENT_WITH_MAIN`,
include doctest. Provides the test binary entry point.

**`tests/test_compile_all.cpp`** — the key test. `#include`s every header in
`libraries/rpdsp/src/rpdsp/`. This is the test that would have caught the
doc-drift class of bug (a header that references a missing type fails to
compile). Body is a single `TEST_CASE` that does trivial work to ensure the
includes are not optimized away.

**`tests/test_algorithm.cpp`** — ~5 smoke tests on pure functions:
- `clamp` / `clamp01` bounds
- `midiNoteToHz(69)` ≈ 440.0
- `dbToGain(0)` ≈ 1.0
- `softClip` shape (0→0, large→asymptote)
- `wrap01` periodicity

**`tests/test_oscillator.cpp`** — ~3 smoke tests:
- `SineOscillator` output within [-1, 1] over a block
- `SawOscillator` output within [-1, 1]
- `Phasor` wraps at 1.0

**Run command:**
```
cmake -S tests -B tests/build && cmake --build tests/build && ctest --test-dir tests/build
```

`tests/build/` is gitignored (added in §1.1).

### 2.2 `scripts/fqbn.txt`

The exact FQBN, lifted from `build/build.options.json` before it is untracked:

```
rp2040:rp2040:rpipico2:flash=4194304_0,arch=arm,freq=200,opt=Optimize3,os=none,profile=Disabled,rtti=Disabled,stackprotect=Disabled,exceptions=Disabled,dbgport=Disabled,dbglvl=None,usbstack=picosdk,ipbtstack=ipv4only,uploadmethod=default
```

Both build scripts read this file rather than hardcoding the FQBN, so a board
change is a one-file edit.

### 2.3 `scripts/build_all_examples.sh` and `.ps1`

Iterate `Examples/*/`, run for each:
```
arduino-cli compile --fqbn <fqbn.txt contents> \
  --library libraries/rpdsp --library libraries/pico_audio_i2s \
  Examples/<name>
```
- Print a per-example pass/fail summary line.
- Exit non-zero if any example fails to compile.
- Do **not** flash — CI has no hardware; flashing stays manual.
- `.sh` for CI + Linux/macOS dev; `.ps1` for Windows dev (matches the user's
  `cmd.exe` / Windows environment).

### 2.4 `.github/workflows/ci.yml`

Triggers: push and PR to `main` and `looped`.

**Job 1 — `host-tests`** (ubuntu-latest):
- Checkout.
- Install CMake + a C++17 compiler (preinstalled on ubuntu-latest).
- `cmake -S tests -B tests/build && cmake --build tests/build && ctest --test-dir tests/build`.
- ~30 seconds. Fast gate.

**Job 2 — `build-examples`** (ubuntu-latest):
- Checkout.
- Install arduino-cli.
- `arduino-cli core install rp2040:rp2040` (downloads the earle-philhower
  core + arm-gcc toolchain, ~hundreds of MB, on first run).
- Run `scripts/build_all_examples.sh`.
- ~3–5 minutes. Slower gate.

CI does **not** flash hardware. It catches compile regressions, API drift, and
broken includes — the failure modes that produced the current doc drift.

**Tuning knob (not decided now):** if `build-examples` proves too slow or
flaky in CI, it can be made a manual `workflow_dispatch` job later. The
host-tests job remains the always-on gate.
